## 1. 产品定位与核心价值

### 1.1 一句话定位
**站在用户这边、会算「个人专属到手价」、能主动盯价、帮你做购买决定、还帮你避开打折陷阱的 AI 购物决策助手——首发手机/3C 数码，微信小程序。**

不做"更好的比价工具"（拼数据深度打不过慢慢买），不做"更好的导购社区"（拼内容打不过什么值得买）。
**做"AI 购物决策代理"——他们卖信息，我们卖决策与省心。**

### 1.2 核心价值（四件事）
1. **算清「个人专属到手价」**：标价 − 平台券 − 店铺券 − 满减 − 补贴 −（用户申报的）会员价/国补 = 真实到手价。
2. **跨平台比价 + 一句话买/等建议**：直接给结论，不堆信息。
3. **主动盯价 + 降价提醒**：把用户留下来、唤回来。
4. **反套路避坑**：识别先涨后降、假补贴、第三方店铺/翻新风险。

### 1.3 核心原则（红线）
- 不做全品类、不做四平台同时首发。
- 不承诺"全网绝对最低价"。
- 不碰用户电商账号 Cookie、不模拟登录、不绕过风控、不爬虫。
- 不让 AI 编造价格（AI 只解释，不算价）。
- CPS 返佣不影响推荐排序，且透明披露。

### 3.1 用户画像
| 画像 | 特征 | 核心痛点 | 对应能力 |
|---|---|---|---|
| A. 价格敏感购机者（22–35，1–30 天换机） | 多平台比价、愿研究补贴但算不清 | 不知道哪买最便宜、是否值得现在买 | 到手价 + 比价 + 买/等建议 |

1. 作为换机用户，输入机型或粘贴链接 → 看到京东/拼多多真实到手价对比 → 知道哪买最划算。

4. 作为纠结"现在买还是等"的用户，看到 AI 的买/等建议 → 减少决策成本。
5. 作为预算有限的用户，设目标价 → 降到价位微信通知我。

## 5. 核心功能详述

### 5.1 省钱资产库（核心创新，破解千人千面）
- 首页提供开关/标签：`88VIP` `京东PLUS` `拼多多省钱月卡` `所在地国补(下拉选省市)`。
- 勾选后全局缓存（本地 + 用户档案），到手价计算时自动叠加对应专属优惠。
- **价值**：不爬用户账号、由用户自申报，即可算出"接近个人专属"的到手价，规避隐藏价拿不到与合规风险。

### 5.2 标准 SKU 识别（数码比价的地基）
- 平台标题 → 标准属性：品牌/系列/型号/存储/颜色/网络版本/国行港版/全新二手翻新/裸机套装。
- 输出：标准 SKU 名 + 置信度 + 风险标签 + 需用户确认字段。
- 置信度规则：
  - **高**：品牌/型号/存储/版本明确。
  - **中**：缺颜色或店铺类型不明确。
  - **低**：疑似套装/二手/翻新/标题规格不完整 → 进人工审核池（单人 MVP 先覆盖热门机型）。

### 5.3 个人专属到手价（规则引擎，非 AI 编价）
- 计算项：`标价 − 平台券 − 店铺券 − 跨店满减 − 平台/官方补贴 −（资产库）会员价/国补 +（运费）`。
- 暂不精确纳入：个人随机红包、支付渠道优惠、直播间专属价、新客专享价、以旧换新（标注"可能更低"）。
- 展示口径：当前参考到手价 / 已纳入优惠 / 未纳入优惠 / 价格更新时间。
- 必显免责声明："价格为公开优惠与平台返回数据估算，最终以下单页为准。"

### 5.4 AI 购买建议（工具调用，防幻觉）
- 输入：商品标题、标准 SKU、当前到手价、近 30/90 天低价、平台、店铺类型、优惠拆解、风险标签。
- 输出：`结论(买/等/谨慎/不建议)` + `≤3 条原因` + `风险提示` + `置信度`。
- 事实数据全部来自工具（见 §9），AI 只负责理解、编排、解释。
- 示例：
  > 结论：谨慎买
  > 原因：当前参考到手价比近 30 天低价高 230 元；该商品为第三方店铺，标题含"未拆封"，需确认发票和官方保修；如不急，建议关注京东自营或拼多多百亿补贴官方渠道。

### 5.5 盯价 + 降价提醒（留存）
- 加入即开始记录价格快照（自有数据资产，从第一天积累）。
- 默认目标价规则：
  - 有 30 天低价 → 默认目标价 = 近 30 天低价。
  - 当前接近低价 → 当前价再降 3%。
  - 历史数据不足 → 当前价再降 5%。
- 刷新频率（漏斗轮询，控成本与限频）：热门 SKU 1–3h / 用户关注 3–6h / 普通每天。
- 触达即发微信订阅消息。

### 5.6 反套路 + 风险标签
- 先涨后降：促销价高于近 N 天常态价 → "⚠️ 折扣可能不实"。
- 风险标签（首版）：第三方店铺 / 非官方旗舰店 / 疑似翻新二手 / 疑似非国行 / 套装不可直接比价 / 规格不全 / 售后不确定 / 价格明显低于常规区间。

## 6. 业务规则（关键防错）

**同款规则**：品牌/型号/容量一致；颜色可合并但标注；定制版/合约版/套餐版不合并；赠品型 SKU 不参与最低价排序。
**到手价规则**：由规则引擎算、非 AI 生成；必须标注前提（会员/券/补贴条件）；超时标注"价格可能过期"；赠品价值不明不计入。
**风险规则**：第三方店必提示；发票/售后不确定需标注；同款置信度不足显示"疑似同款，需谨慎比较"。
**中立规则**：CPS 不影响推荐排序；有返佣需透明披露；敢于输出"不建议买"。

- 链接为空：请粘贴京东或拼多多手机商品链接。
- 平台不支持：当前版本优先支持京东和拼多多。
- 非手机商品：该商品暂不在手机品类范围内。

### 7.3 商品分析页（核心转化页，顺序）
1. 商品基础信息（主图/标题/平台/店铺/规格）

5. 风险提示（店铺类型/版本风险/套装风险/售后风险）
6. 历史价格（30/90 天趋势，数据不足时明确提示"历史价格积累中"）
7. 同款对比（京东与拼多多同款或近似 SKU）

## 9. 技术架构（单人 Android + AI 可维护）

### 9.1 架构图
```
微信小程序(原生) → API 网关/BFF →
[用户服务][商品服务][价格服务][SKU解析服务][AI分析服务][盯价服务][通知服务][后台]
            ↓                         ↓                ↓
   联盟API适配层(多多进宝/京东)   大模型API(工具调用)   价格快照库 + 定时任务(Quartz)
            ↓
        Redis 缓存(券/价热数据，控成本与限频)
PostgreSQL(业务) + 价格快照(时序) + Redis
```

### 9.2 技术选型（贴合 Android 工程师 + 单人）
- **前端**：微信小程序**原生**（最快上线，剪贴板/订阅消息/跳转能力最直接）。后续如需扩 App，再考虑 Flutter（Android 背景上手快）。
- **后端**：**Kotlin/Java + Spring Boot 3**（吃你的 Android 底子）；**先模块化单体**，按服务边界分包，用量上来再拆部署。
- **数据库**：PostgreSQL（业务 + 价格快照）+ Redis（缓存/限流）。
- **定时任务**：Quartz / Spring Scheduling。
- **部署**：Docker + 单台云服务器起步。

### 9.3 AI 层（工具调用，防幻觉）
模型不编价，事实全部走工具（Function Calling）：
- `parse_link(text)` → 平台/itemId/原始商品
- `search_products(...)` → 候选商品
- `get_best_price(sku, userAssets)` → 个人到手价
- `get_price_history(sku)` → 历史价
- `detect_risk(item)` → 风险标签
- `suggest_addon(amount, rule)` → 凑单（P2）

**模型选型建议**：复杂促销规则/SKU 解析用高精度模型；常规计算/代码生成用便宜快的模型。
**成本控制**：热门 SKU 结果缓存；结构化输入避免长文本；高置信度走规则、低置信度才调模型；AI 输出必须可追溯到价格事实。

### 9.4 关键工程点
- 联盟 API 限频 → 缓存 + 批量 + 错峰（漏斗轮询）。
- 价格快照第一天就攒。
- 到手价计算服务独立、纯规则。

## 11. 接口设计（核心，带出入参）

### 11.1 链接解析
`POST /api/link/parse`
- 入参：`link_text`
- 出参：`platform` / `item_id` / `product_raw` / `parse_status` / `error_message`

### 11.2 商品分析结果
`GET /api/product/analysis?platform&item_id&assets`
- 出参：`product_info` / `sku_info` / `price_info` / `trend_info` / `risk_info` / `ai_recommendation`

## 13. 商业模式

- **主营 CPS 返佣（首期）**：手机单均 GMV ~¥2000，佣金 1%–3% → 单笔 ¥20–60；家电更高。高客单使少量订单即可验证闭环。
- **会员（P2）**：更多提醒名额 / 历史价趋势 / AI 深度分析 / 大促预警；¥9.9/月 或年付。
- **远期**：B 端价格监控/行情报告、品牌合规合作。
- **中立承诺**：返佣不影响排序、透明披露——这是相对慢慢买/什么值得买的信任壁垒。

## 15. 验收标准

1. **链接查询**：给定京东/拼多多手机链接，点击查询后展示商品信息、参考到手价、风险标签、AI 建议。
2. **到手价**：展示价、可用公开券、补贴、估算到手价、更新时间齐全；含资产库时叠加专属优惠。
3. **AI 建议**：给定价格/历史价/平台/店铺/风险标签，输出明确结论 + ≤3 条理由 + 置信度。
4. **关注提醒**：用户设目标价，价格低于目标价时推送微信订阅消息。
5. **合规说明**：分析页必含参考价说明，不得出现"全网绝对最低价"等绝对化承诺。
6. **首版成功指标（上线 30 天）**：真实查询用户 1000+；解析成功率 70%+；结果页停留 30s+；查询后关注比例 15%+；7 日重复查询 20%+；建议有帮助反馈 60%+。

## 16. 风险与对策
