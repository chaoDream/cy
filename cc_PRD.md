## 1. 产品背景与问题

### 1.1 背景
电商平台的价格体系日益复杂：满减、跨店、平台券、品类券、百亿补贴、会员价、先涨后降……普通用户**算不清"到底哪家、用什么券、什么时候买最便宜"**，且容易掉进"假折扣"陷阱。

### 1.2 用户的真实痛点
1. **不知道现在是不是低价**：怕刚买就降价、怕错过真低点。
2. **算不清到手价**：券太多太乱，平台显示价 ≠ 实际到手价。
3. **凑单痛苦**：满300减50，差几十块不知道加买什么最划算。
4. **怕被套路**：先涨后降、伪百亿补贴、规格缩水难以辨别。
5. **决策累**：信息过载，要在多个 App/页面之间反复比对。

### 1.3 现有方案的不足（竞品软肋）
| 竞品 | 本质 | 护城河 | 软肋（我们的机会） |
|---|---|---|---|
| 慢慢买 | 价格工具 | 10年+历史价数据库 | 只给图表不替你决策；体验偏工具、偏老旧 |
| 什么值得买 | 内容社区/导购媒体 | 内容生态+社区+流量 | 信息过载需自己刷；广告化重，用户需自行甄别 |

**核心洞察：两者都是"信息提供者"——把数据丢给你，让你自己判断。AI 时代用户要的不是更多信息，而是更少的决策负担。**

## 2. 产品愿景与策略

### 2.1 愿景
让每一次网购都"买得明白、买得最优、不被套路"，把购物决策的脑力活交给 AI。

### 2.2 核心策略：换价值轴
- **不做**"更好的比价工具"（拼数据深度打不过慢慢买）。
- **不做**"更好的导购社区"（拼内容生态打不过什么值得买）。
- **要做**："决策代理（Agent）"——他们卖**信息**，我们卖**决策与省心**。

### 2.3 四个差异化支点
1. **从"给信息"→"给答案 + 帮你下手"**：对话式购物决策，直接给 1 个推荐 + 理由 + 该不该现在买。
2. **从"被动查"→"主动替你干活"**：自动盯价、囤货提醒、智能凑单。
3. **旗帜鲜明站在用户这边**：反套路、打假折扣、防智商税。
4. **垂直深耕**：先把一个品类做穿，建立认知壁垒，再横向扩张。

## 3. 切入品类选择：母婴（首选）

### 3.1 候选品类评估
| 评估维度（权重） | 母婴 | 3C数码 | 美妆 |
|---|---|---|---|
| 复购/高频（强相关留存） | ★★★★★ | ★★ | ★★★★ |
| 决策复杂度（AI 价值） | ★★★★ | ★★★★★ | ★★★★ |
| 付费/价格敏感意愿 | ★★★★★ | ★★★★ | ★★★★ |
| 口碑/社群传播性 | ★★★★★ | ★★★ | ★★★★★ |
| 信任敏感（反套路定位契合） | ★★★★★ | ★★★ | ★★★★★ |
| 数据可得性（联盟API） | ★★★★ | ★★★★ | ★★★★ |
| **综合** | **最优** | 偏低频留存差 | 次优（可作二期） |

### 3.2 为什么是母婴
- **超高复购**：奶粉、尿不湿是月度刚需，天然驱动用户**反复回来**——这是工具类产品最稀缺的留存抓手。
- **计划性囤货**：妈妈们囤货习惯强，"盯价 + 囤货提醒 + 凑单"场景天然契合。
- **价格敏感 + 开销大**：母婴是家庭重支出，省钱诉求强、肯用工具。
- **传播性极强**：妈妈群、宝妈社群分享意愿高，低成本裂变获客。
- **信任敏感**：母婴对安全/真伪/规格极度敏感，"站在用户这边、反套路"的定位特别能圈住人心。

### 3.3 风险提示
- 奶粉等品类有监管/真伪要求 → MVP 阶段**只做比价/导购/决策，不碰商品真伪背书与跨境合规承诺**，相关由平台与品牌方负责，我们只做"价格与决策辅助"。

### 3.4 子品类优先级（母婴内部）
- **P0**：尿不湿、奶粉、湿巾、纸尿裤（高频快消，盯价/囤货/凑单价值最大）
- **P1**：辅食、洗护、喂养用品
- **P2**：玩具、出行（推车安全座椅）、服饰

## 4. 目标用户

### 4.1 用户画像
- **核心用户**：25–38 岁宝妈（孕期至宝宝 3 岁），家庭月母婴支出 1000–3000 元，价格敏感、爱比价、活跃于宝妈社群。
- **次要用户**：准爸爸、隔代育儿的长辈代购、母婴囤货党。

### 4.2 典型使用场景
1. **日常补货**："尿不湿快用完了，现在哪家买最划算？"
2. **囤货决策**："这款奶粉现在是低点吗？要不要囤 3 罐？"
3. **大促作战**："618 我列了一车东西，帮我算最优券组合和下单时间。"
4. **凑单**："差 40 块满减，加买什么最值？"
5. **避坑**："这个所谓的百亿补贴是真便宜还是先涨后降？"

## 5. MVP 范围定义

### 5.1 MVP 目标（验证什么）
**核心假设**：宝妈愿意用一个"能直接给购买建议 + 帮忙凑单 + 主动盯价"的 AI 工具，并通过它的链接下单（产生 CPS 佣金）。

**验证指标（北极星）**：**通过省心买完成下单跳转的次数 / 周活用户**（即"决策→行动"转化）。

### 5.2 做什么（In Scope）
- 母婴品类的**比价 + 补贴后到手价测算**
- **对话式购物决策**（自然语言问需求 → 给推荐 + 理由 + 买点建议）
- **智能凑单**助手
- **自动盯价 + 囤货提醒**
- **反套路识别**（先涨后降/假折扣提示，基于自建价格快照）
- **CPS 转链下单跳转**
- 首发平台：**多多进宝（拼多多）** + 淘宝联盟/京东联盟基础 API

### 5.3 不做什么（Out of Scope，MVP 阶段）
- ❌ 全品类覆盖（只做母婴）
- ❌ 自营商城/交易闭环（只做导购跳转，不碰交易/资金）
- ❌ 社区 UGC/内容论坛（不与什么值得买正面竞争）
- ❌ "全网绝对最低价"承诺（合规 + 口碑风险，改为"已知可领券与补贴后的最优价"）
- ❌ 爬虫硬抓平台页面（法律红线，仅用官方联盟 API）
- ❌ 商品真伪/跨境合规背书
- ❌ App（一期只做微信小程序）

## 7. 核心功能详述

### 7.1 对话式购物决策 Agent（P0，核心差异点）

**用户故事**：作为一名宝妈，我想用一句话说出需求，让 App 直接告诉我买哪个、为什么、现在该不该买，而不用自己刷一堆页面。

**交互流程**：
1. 用户输入自然语言：如"宝宝 6 个月，想买拉拉裤，预算每片 1 块以内，要透气"。
2. Agent 解析意图，缺关键信息时**最多反问 1–2 个问题**（尺码/数量/品牌偏好）。
3. 跨平台检索候选商品 → 计算补贴后到手价 → 结合价格快照判断价位水平。
4. 输出：**1 个主推 + 1–2 个备选**，每个含：到手价、关键卖点、推荐理由、"现在买/再等等"建议、下单按钮。

**设计原则**：少而准（不堆列表）、有理由（可解释）、给行动（直接能下单）。

### 7.2 比价与"补贴后到手价"（P0）

- 数据来源：多多进宝 API（含百亿补贴频道）为主，淘宝/京东基础 API 补充。
- 计算：`到手价 = 商品价 − 可叠加的公开券 − 平台补贴（接口可得部分）`。
- **诚实标注**：明确标注"基于已知可领券与公开补贴测算，个性化隐藏券可能更低"。

### 7.3 智能凑单（P0，不依赖隐藏券的差异化亮点）

**问题**：满减门槛差一点，用户不知道加买什么。

**方案**：
- 输入：当前商品/购物车金额、满减规则、品类偏好。
- 算法：在"母婴高复购刚需品"池中，求解**最接近且不浪费**的凑单组合（背包/贪心近似），优先推荐用户本来就要囤的快消品（尿不湿/湿巾/奶粉）。
- 输出：凑单清单 + 凑后实付 + 每件折后单价 + 一键加购跳转。

### 7.4 自动盯价 + 囤货提醒（P0/P1）

- 加入心愿单即开始**记录价格快照**（这是我们从第一天积累的自有数据资产）。
- 触发条件：到达用户设定目标价 / 创历史快照新低 / 大促开始。
- **囤货提醒（P1）**：基于品类复购周期（如尿不湿约 3–4 周）主动提示"该补货了，且当前价合适"。

### 7.5 反套路识别（P0/P1）

- 基于自建价格快照，识别**先涨后降**：若"促销价"高于近 N 天常态价，标注"⚠️ 这个折扣可能不实"。
- 假百亿补贴提示（P1）：与常态价对比给出真实降幅。
- **定位价值**：建立"站在用户这边"的信任品牌。

## 9. 数据来源与合规

### 9.1 数据来源（仅官方联盟 API，杜绝爬虫）
| 平台 | 接入身份 | 可得数据 | 接入难度 |
|---|---|---|---|
| **多多进宝（首发）** | 多多进宝推手（个人/企业均可） | 商品库、百亿补贴频道、券、CPS锁佣 | 低（最友好） |
| 淘宝联盟 | 媒体备案 + 基础API | 商品搜索/详情、公开券、选品库 | 中（高级API邀请制，暂不指望） |
| 京东联盟 | 企业账号 + 备案 + 基础API | 商品、公开券、转链、订单 | 中（高级API需流量证明） |
| 抖音（穿山甲/抖客） | 独立抖客 | 商品、转链、订单 | 中（二期） |

### 9.2 合规红线
1. **绝不爬取**平台页面数据（已有大量"不正当竞争/数据侵权"判例）。
2. **不承诺**"全网绝对最低价"，统一表述为"已知可领券与公开补贴后的最优价"。
3. 价格快照仅来源于合法接口与用户查询，标注更新时间。
4. 企业主体尽早注册（京东/拼多多高级权限与对公结算需要）。

## 10. 技术架构建议

> 结合你的技术栈（Java/Kotlin/Compose/Flutter/RN），一期最省力的是小程序 + 轻后端。

### 10.1 总体架构
```
[微信小程序前端]
        │ HTTPS
        ▼
[API 网关 / BFF]
        │
   ┌────┼─────────────┬───────────────┐
   ▼    ▼             ▼               ▼
[商品聚合服务]  [AI 决策服务]   [盯价/提醒服务]   [用户/账户服务]
   │              │                │
   ▼              ▼                ▼
[联盟API适配层]  [大模型API]   [价格快照库 + 定时任务]
(多多进宝/淘/京)              (时序数据)
   │
   ▼
[缓存层 Redis]（券/价格热数据，控成本与限频）
```

### 10.2 技术选型建议
- **前端**：微信小程序原生 或 Taro（可复用到多端）；二期 App 用 **Flutter**（你已精通，跨 iOS/Android 省力）。
- **后端**：Spring Boot（Java/Kotlin，你最熟）或 Go；BFF 聚合多平台数据。
- **数据库**：PostgreSQL（业务） + 时序/快照表（价格历史） + Redis（缓存联盟数据，规避限频与降本）。
- **定时任务**：盯价轮询、快照采集（注意联盟 API 限频）。
- **AI 层**：调用大模型 API（见第 11 节），用 RAG/工具调用把"商品检索+到手价计算"接给模型。

### 10.3 关键工程注意
- **联盟 API 限频**：用缓存 + 批量 + 错峰，避免触发风控/限流。
- **价格快照从第一天积累**：这是长期数据护城河。
- **到手价计算服务独立**：规则多变，单独模块便于维护。

## 11. AI 能力设计

### 11.1 模型职责
- **意图理解 + 多轮澄清**：把自然语言需求结构化为检索条件。
- **推荐解释**：把"价格 + 参数 + 快照"组织成人话理由。
- **买点建议**：结合快照给"建议入手/再等等"。

### 11.2 架构：工具调用（Function Calling / Agent）
模型不直接"编造"价格，而是通过工具拿真实数据：
- `search_products(品类, 预算, 属性...)` → 联盟 API
- `get_best_price(商品ID)` → 到手价计算服务
- `get_price_history(商品ID)` → 价格快照库
- `suggest_addon(金额, 满减规则)` → 凑单算法

模型负责"理解 + 编排 + 解释"，**事实数据全部来自工具**，规避幻觉与价格错误。

### 11.3 成本控制
- 简单意图用小模型/规则，复杂决策才用大模型。
- 缓存常见问答与商品摘要。
- 控制上下文长度，结构化输入而非塞原始数据。

## 12. 商业模式

- **主要变现**：CPS 佣金（用户经省心买链接下单，平台返佣）。
- **可选**：把部分佣金以"返现"返给用户，增强黏性（P2）。
- **远期**：垂直品牌的精准推广位、会员订阅（更强盯价/比价能力）。
- **不碰**：自营交易、资金沉淀（合规与重资产风险）。

| 风险 | 等级 | 对策 |
|---|---|---|
| 隐藏券/个性化价拿不到，"最低价"不准 | 高 | 诚实定位"已知可领券后的最优价"；强化凑单/盯价/反套路等不依赖隐藏券的价值 |
| 联盟高级 API 门槛高（淘/京需流量） | 中 | 首发走多多进宝；用基础 API 起步；有流量后再申请 |
| AI 给出错误价格/推荐（幻觉） | 中 | 事实数据全走工具调用，模型不编价；到手价人工抽检 |
| 冷启动无历史价数据 | 中 | 第一天起自建快照；用 AI 常识 + 当前价做决策，不依赖深度历史 |
| 获客成本 | 中 | 小程序裂变 + 宝妈社群 + 小红书/抖音内容，不烧硬广 |
| 巨头跟进 | 中 | 垂直深耕建认知 + 用户个性化数据积累，做大公司转身慢的体验 |
| 合规（爬虫/虚假宣传） | 高 | 只用官方 API；不承诺绝对最低价；不碰真伪背书 |

## 18. P0 功能验收标准（Acceptance Criteria）

> 采用 Given / When / Then 描述，作为后续开发与测试的判定依据。

### 18.1 对话式决策
- **Given** 用户输入一句含品类+预算的需求，**When** 信息足够，**Then** 系统在 1 轮内返回 1 个主推 + ≤2 个备选，每项含到手价、理由、买点建议、下单入口。
- **Given** 用户需求缺关键信息（如尺码），**When** 解析后无法定位，**Then** 系统最多反问 1–2 个问题，不无限追问。
- **Given** 检索无合适结果，**When** 候选为空，**Then** 明确告知"暂未找到合适商品"并给出放宽条件建议，不编造商品。

### 18.2 比价 / 到手价
- **Given** 一个商品，**When** 展示详情，**Then** 显示商品价、可叠加公开券、测算到手价、数据更新时间，并标注"基于已知可领券测算"。
- **Given** 多平台有同款，**When** 比价，**Then** 按到手价升序展示并标注各平台。

### 18.3 智能凑单
- **Given** 当前金额与满减规则，**When** 请求凑单，**Then** 返回凑单清单使实付跨过门槛且溢出最小，优先母婴高复购品，并给凑后实付与单价。

### 18.4 盯价 / 心愿单
- **Given** 用户加入心愿单，**When** 加入成功，**Then** 系统开始记录该商品价格快照。
- **Given** 商品达到目标价或创快照新低，**When** 触发，**Then** 在设定渠道推送提醒（含当前价与建议）。

### 18.5 反套路
- **Given** 某商品"促销价"高于近 N 天常态价，**When** 展示，**Then** 标注"⚠️ 该折扣可能不实"并给出近期价格参考。

### 18.6 CPS 跳转
- **Given** 用户点击下单，**When** 跳转，**Then** 通过合法转链跳转对应平台且可被联盟正确归因。

## 19. 关键用户旅程（User Journeys）

1. **日常补货**：首页对话"尿不湿快没了，哪家划算" → Agent 给主推+到手价 → 用户点下单跳转 → （可选）加入心愿单盯价。
2. **囤货决策**：搜某奶粉 → 详情页看快照与"是否低点" → Agent 建议"建议囤2罐"或"再等等" → 凑单到满减 → 下单。
3. **大促作战（远期强化）**：促销前把多商品加入心愿单 → 大促开始推送 → 一键查看最优券组合与下单顺序。

## 20. 术语表 / 数据字典

| 术语 | 含义 |
|---|---|
| 到手价 | 商品价扣除可叠加公开券与接口可得补贴后的近似实付价 |
| 价格快照 | 系统在某时刻记录的商品价格，用于历史比较与反套路（自有数据资产） |
| 心愿单 | 用户收藏并触发盯价的商品集合 |
| PID / 转链 | 联盟推广位标识 / 带归因参数的下单链接，CPS 结算依据 |
| CPS | 按成交付费（Cost Per Sale），用户下单后平台返佣 |
| 推手 | 多多进宝中第三方推广者身份（个人/企业均可） |
| 常态价 | 某商品近 N 天非促销的基准价，用于识别"先涨后降" |

## 21. 待决策的开放问题（需你拍板）

> 这些直接影响产品形态与商业模式，建议优先确认。

1. **是否做"返现"**（把部分 CPS 佣金返给用户）？利于获客留存，但压缩利润。MVP 做不做？
2. **首发平台范围**：只做多多进宝先验证，还是一上来就接淘宝/京东基础 API？（影响开发量与冷启动数据丰富度）
3. **登录方式**：纯微信授权，还是需要绑定手机号？（影响转化率与召回能力）
4. **反套路冷启动**：自有快照不足时，是否短期"不下结论"，还是引入有限的第三方/人工兜底？
5. **选品策略**：完全算法/AI 驱动，还是 MVP 期加入人工精选兜底保证推荐质量？
6. **商业边界**：是否完全不碰商品真伪/品牌背书？（合规更安全，但用户可能期待"靠谱"信号）
7. **目标人群再聚焦**：是否进一步收窄到"0–1 岁新手妈妈"，做更极致的冷启动？

## 22. 自评：可优化点与权衡（供 Review）

> 我对本文档主动挑出的潜在弱点，供你判断是否调整。

1. **"到手价准确率"是命门但最难**：隐藏券拿不全会让核心卖点打折。可考虑把 MVP 主打卖点从"最低价"进一步前移到**"凑单 + 盯价 + 反套路"**这些我们能 100% 做好的能力，"到手价"作为辅助。
2. **对话式 Agent 开发成本可能偏高**：若资源紧张，MVP 可先做"结构化筛选 + 一句话建议"的轻量版，把完整多轮对话放到 P1，降低首版风险。
3. **北极星指标可能过早偏向变现**：冷启动期或许更应先看"留存/盯价提醒打开率"，等留存稳了再优化下单转化。建议设主/辅两个北极星。
4. **母婴真伪敏感**与"不碰真伪背书"存在张力：用户在母婴上恰恰最在意靠谱。需想清楚如何在不背书的前提下传递"可信"信号（如只推官方旗舰店/百亿补贴货源）。
5. **冷启动数据飞轮慢**：自建快照需要时间。可考虑首发期对少量高频 SKU（如 Top 50 尿不湿/奶粉）做高频快照，集中把这几个单品的"低点判断"做准，而非全品类铺开。
6. **缺少明确的竞品功能对照清单**：可补一张"省心买 vs 慢慢买 vs 什么值得买"的逐功能对照表，让差异化更可量化。
7. **缺少 UI/交互原型**：本版为纯文字需求，下一步建议补关键页面线框图，避免开发期理解偏差。
